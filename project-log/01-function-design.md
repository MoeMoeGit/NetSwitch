## 模块一：方案管理

### 功能列表

- [x] 内置 `DHCP（默认）` 方案，固定 `id=default`，不可删除和重命名。
- [x] 用户可新建、编辑、重命名、删除方案。
- [x] 首次启动时自动创建 `%AppData%\NetSwitch\profiles.json`。
- [x] 首次启动如果检测到当前网卡为静态 IP，则导入为「自定义」方案并设为激活。
- [x] 记录 `active_profile_id`、`last_used`、窗口坐标和设置项。

### 业务逻辑

```text
启动 → load_config()
  ├─ 配置文件不存在 → 创建默认配置 → 检测当前网络 → 如为静态 IP 则追加「自定义」方案 → 保存
  └─ 配置文件存在 → 读取 JSON → 补齐默认方案 → 返回配置
```

### 边界情况

- 配置 JSON 损坏时会先备份为 `profiles.invalid.<timestamp>.json`，再重建默认配置。
- 当前实现编辑 DHCP / 自动 DNS 时会清理旧的静态字段，避免展示混乱。
- 配置文件不包含 `adapter` 字段，切换时总是实时识别当前优先网卡。

## 模块二：网络切换

### 功能列表

- [x] 通过 `route print 0.0.0.0` 获取当前默认路由对应 IP。
- [x] 通过 PowerShell 查询网卡别名、IP、网关、DNS。
- [x] 通过 `netsh interface ip set address` 切换 DHCP / 静态 IP。
- [x] 通过 `netsh interface ip set/add dns` 切换 DNS。
- [x] 切换后 ping 网关，返回成功、网关不通或失败。
- [x] 写入后等待改为最多 5 秒轮询，每 500ms 检测是否生效。

### 业务逻辑

```text
用户选择方案 → 读取当前优先网卡 → 读取旧配置
  → 执行 IP 写入
  → 执行 DNS 写入
  → 等待生效（最多 5 秒轮询）
  → ping 网关
  → 更新 active_profile_id / last_used / 托盘状态
```

### 边界情况

- 当前实现没有真正按 metric 排序默认路由，只取解析到的第一条默认路由；多网卡 / VPN 场景可能选错。
- 当前实现部分失败路径会进入统一回滚，DNS 写入失败也会返回详细错误。
- 当前实现静态 IP 强制要求网关；如果后续支持无网关隔离网段，需要补充产品和实现决策。

## 模块三：主界面

### 功能列表

- [x] 固定宽度 340px，不可缩放。
- [x] 标题栏包含最小化和关闭按钮，关闭表示隐藏到托盘。
- [x] 状态栏显示当前 IP 和网关。
- [x] 卡片列表展示方案名称、副文本、激活状态、选中状态。
- [x] 底部按钮：新建、删除、激活。
- [x] 键盘快捷键：Enter 激活、Delete 删除、F2 重命名、ESC 取消选中或关闭。
- [x] 窗口位置记忆，恢复前校验坐标是否在屏幕范围内。

### 业务逻辑

```text
打开主界面 → load_cards()
单击卡片 → 选中
点击激活 / Enter → 后台线程应用方案 → 成功后更新配置和卡片
双击卡片 → 打开编辑弹窗
右键卡片 → 激活 / 重命名 / 查看详情 / 删除
```

### 边界情况

- 当前实现激活成功后会重新选中激活卡片；原设计曾要求清除选中状态，此差异暂按当前实现记录。
- 状态栏刷新在 UI 线程中执行 PowerShell / ping 相关读取，可能造成卡顿。
- 删除当前激活方案时先切回默认 DHCP，成功后再删除；失败则阻止删除。

## 模块四：托盘与设置

### 功能列表

- [x] 托盘右键菜单展示所有方案，当前激活方案打勾。
- [x] 托盘菜单支持直接切换方案。
- [x] 托盘菜单提供「打开主界面」「开机自启」「设置」「退出」。
- [x] 左键单击托盘图标打开主界面。
- [x] 设置弹窗包含「开机自启」和「开机恢复上次方案」。
- [x] 设置弹窗包含「检查更新」和「帮助与反馈」。
- [x] 勾选设置后立即写入 `profiles.json`。

### 业务逻辑

```text
托盘选择方案 → 后台线程 apply_profile() → 更新托盘状态 → 更新 active_profile_id / last_used
托盘开机自启 → 写 HKCU Run 注册表 → 写 profiles.json
设置弹窗开机恢复 → 写 profiles.json
设置弹窗检查更新 → 请求 GitHub Release → 有新版本时可下载并运行安装包
```

### 边界情况

- 托盘连续点击多个方案时有全局互斥保护，避免并发执行多个网络写入。
- 开机自启注册表写入失败会返回错误并恢复 UI 状态，路径已显式加引号。
- 单实例唤起依赖可见窗口枚举，但主窗口隐藏后仍保留引用，当前可恢复已有窗口；后续如改造窗口生命周期仍需复核。
