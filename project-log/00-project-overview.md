## 项目名称

NetSwitch

## 项目背景

NetSwitch 是一个 Windows 11 系统托盘网络配置切换工具，用于在 DHCP 与多套静态 IP / 网关 / DNS 方案之间快速切换。目标用户不需要进入 Windows 系统设置手动修改网络参数。

核心目标：

- 简单快捷：托盘右键或主界面即可切换方案。
- 资源占用低：常驻托盘，不做复杂后台服务。
- 安全可恢复：切换前读取旧配置，失败时应回滚。

## 用户 / 使用场景

- 笔记本每天在家庭、公司、客户现场之间移动，需要切换 DHCP / 静态 IP。
- 调试软路由、自建 DNS、旁路网关时，需要频繁更换网关和 DNS。
- 在隔离网络、客户现场网络或临时实验环境中，需要快速恢复到默认 DHCP。

## 核心功能

1. 系统托盘常驻，托盘右键菜单直接切换网络方案。
2. 主界面管理多套网络配置方案，支持新建、编辑、重命名、删除和激活。
3. 每次切换时自动检测当前优先网卡，不在配置文件中保存网卡字段。
4. 切换前读取当前配置，切换失败时设计上应自动回滚。
5. 托盘图标和主窗口状态栏显示当前网络状态。
6. 可配置开机自启和开机恢复上次方案。
7. 首次启动自动检测当前网络配置；如果是静态 IP，则导入为「自定义」方案。

## 核心概念

- **方案 Profile**：一套网络配置，包括 IP 模式、静态 IP 参数、DNS 模式、DNS 地址和备注。
- **默认方案**：内置 `DHCP（默认）`，`id` 固定为 `default`，不可删除、不可重命名。
- **当前优先网卡**：程序通过 `route print 0.0.0.0` 读取默认路由，再结合 PowerShell 查询对应网卡名称。
- **激活方案**：配置文件中的 `active_profile_id`。注意它表示程序上次成功应用的方案，不一定能证明系统当前配置仍完全一致。

## 技术栈

| 层级 | 技术 | 说明 |
|------|------|------|
| 语言 | Python 3.12 | `pyproject.toml` 要求 `>=3.12,<3.13` |
| GUI | PyQt6 | 主窗口、弹窗、托盘菜单、动态图标绘制 |
| 网络读取 | `route print` + PowerShell | 读取默认路由、网卡别名、IP、网关、DNS |
| 网络写入 | `subprocess` 调用 `netsh` | 设置 DHCP / 静态 IP / DNS |
| 配置存储 | JSON 文件 | `%AppData%\NetSwitch\profiles.json` |
| 图标生成 | Qt + Pillow | 运行时状态图标用 `QPixmap/QPainter`；构建期 ico/png 用 Pillow |
| 打包 | PyInstaller 单文件 exe | 通过 manifest 声明管理员权限 |
| 安装包 | Inno Setup 6 | 生成 Windows 安装包 |

## 项目边界

- 不提供服务端、Web API 或远程同步。
- 不保存网卡字段，不提供网卡绑定或手动选择网卡。
- 不做全局快捷键监听，快捷键只在主窗口激活时生效。
- 不动态提权；权限由 `app.manifest` 的 `requireAdministrator` 处理。
- 不在主 README 暴露内部评审、开发日志和设计取舍；这些内容保存在 `project-log/`。

## 项目约束

- 目标系统为 Windows 10/11，主要面向 Windows 11。
- 修改网络配置需要管理员权限。
- 网络命令和状态检测可能受系统语言、PowerShell 输出、VPN / 多网卡 / 虚拟网卡影响。
- `project-log/` 是本地开发知识库，当前已被 `.gitignore` 忽略。
